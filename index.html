핵심 요구사항 분석

  • Websocket 기반 실시간 호가 수집 — 빗썸, 업비트, Gate.io, Binance, Bybit 5개 거래소의 매수 1호가 / 매도 1호가를 Websocket으로 실시간 수신
  • 스프레드 계산 및 필터링 — 거래소 간 가격 차이를 실시간 계산하고, 사용자가 설정한 최소 수익률 이상의 결과만 필터링하여 표시
  • 입출금 상태 필터 — 각 거래소·코인별 입금/출금 가능 상태를 조회하고, 출금 중지 코인은 모니터링 대상에서 자동 제외
  • 네트워크 정보 표시 — 동일 코인이라도 거래소별로 지원하는 네트워크(ERC-20, TRC-20, BEP-20 등)가 다를 수 있으므로 명확히 표시
  • Gate.io 전용 대출 가능 여부 — Gate.io 거래소에 한해 모니터링 대상 코인의 마진 대출 가능 여부를 조회·표시
  • 알림 기능 — 설정 수익률 초과 시 소리 알림 + 텔레그램 메시지 알림을 즉시 발송

제안 포인트

본 프로젝트는 자동 매매 기능 없이 차익 거래 판단 정보를 신속하게 제공하는 모니터링 시스템입니다. 핵심은 5개 거래소의 Websocket API를 안정적으로 병렬 연결하고, 실시간으로 스프레드를 계산하는 아키텍처 설계에 있습니다.

  • 어댑터 패턴 기반 거래소 모듈화 — 거래소별 API 포맷 차이를 추상화하여 확장성 확보. 향후 거래소 추가 시 어댑터만 추가하면 되는 구조
  • asyncio 기반 비동기 아키텍처 — 5개 거래소 Websocket을 단일 이벤트 루프에서 효율적으로 처리. CPU 자원 최소화
  • 유사 프로젝트 경험 — Python 기반 금융 플랫폼(P2P 크라우드펀딩, 15+ 외부 API 연동) 및 Web3/DeFi 토큰 스왑 플랫폼(Life3) 개발 경험 보유
  • 입출금 상태 + 네트워크 정보 통합 — 단순 가격 비교가 아닌, 실제 차익거래 실행 가능 여부(출금 가능, 네트워크 일치)까지 종합 판단 가능한 시스템

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

국내 및 해외 주요 암호화폐 거래소 간에 발생하는 현물 가격 차이(스프레드)를 실시간으로 감지하고 사용자에게 알려주는 모니터링 시스템을 구축합니다. 본 프로젝트는 자동 매매 기능을 포함하지 않으며, 사용자가 수동으로 차익 거래를 실행하기 위한 판단 정보를 신속하게 제공하는 데 목적이 있습니다.

2. 사용자 정의

사용자 유형설명주요 기능
트레이더 (일반 사용자)국내외 거래소 간 차익 거래 기회를 모니터링하는 사용자실시간 스프레드 조회, 수익률 필터 설정, 알림 수신, 입출금 상태 확인

3. 기능 요구사항

3.1 Websocket 기반 실시간 호가 수집

  • [필수] 빗썸, 업비트, Gate.io, Binance, Bybit 5개 거래소 Websocket 연결
  • [필수] 각 거래소 매수 1호가(best bid) 및 매도 1호가(best ask) 실시간 수신
  • [필수] 연결 끊김 시 자동 재연결 (exponential backoff)
  • [권장] 거래소별 연결 상태 모니터링 및 로그

3.2 스프레드 계산 및 필터링

  • [필수] 모든 거래소 쌍(pair)에 대해 실시간 스프레드 계산
  • [필수] 사용자 설정 최소 수익률(%) 이상만 필터링하여 표시
  • [필수] 매수 거래소 → 매도 거래소 방향별 수익률 표시
  • [권장] 전송 수수료 반영 순수익률 계산

3.3 입출금 상태 필터

  • [필수] 각 거래소·코인별 입금/출금 가능 상태 조회 및 표시
  • [필수] 출금이 중지된 코인은 모니터링 대상에서 자동 필터링
  • [권장] 상태 변경 시 알림 (출금 재개 등)

3.4 네트워크 정보 표시

  • [필수] 동일 코인의 거래소별 지원 네트워크 종류 표시 (ERC-20, TRC-20, BEP-20 등)
  • [필수] 송금 가능한 공통 네트워크 하이라이트

3.5 Gate.io 전용 기능

  • [필수] Gate.io 마진 대출 가능 여부 조회 및 표시
  • [권장] 대출 가능 수량 및 이자율 표시

3.6 알림 기능

  • [필수] 설정 수익률 초과 시 소리 알림 (로컬 사운드 재생)
  • [필수] 텔레그램 Bot API를 통한 메시지 알림
  • [필수] 알림 메시지에 코인명, 매수/매도 거래소, 스프레드, 수익률 포함
  • [권장] 알림 쿨다운 설정 (동일 기회 중복 알림 방지)

4. 비기능 요구사항

항목요구사항

보안API 키 환경 변수 관리, 설정 파일 암호화 권장
호환성Windows / Linux / macOS 크로스 플랫폼 지원

5. 기술 제약사항

  • Python을 주력 언어로 사용 (클라이언트 요구사항)
  • asyncio 또는 멀티쓰레드 구조 활용 필수
  • 각 거래소 Websocket API 사양에 따른 메시지 포맷 처리
  • 거래소 API Rate Limit 준수

6. 범위 외 사항 (Out of Scope)

  • 자동 매매 / 트레이딩 봇 기능
  • 웹 UI / 대시보드 (별도 협의 시 추가 가능)
  • 선물/마진 거래 스프레드 모니터링
  • 거래소 계정 관리 및 자산 조회
  • 과거 데이터 분석 / 백테스팅
